Docker & Kubernetes, fondamentaux 21 Objets K8s : PODS • La configuration d’un POD se fait par le biais d’un Manifest qui est un YAML (ou parfois JSON) comprenant 4 sections : • apiVersion : la version • kind : Pod (d’autres type existent comme Deployment, ReplicaSet, etc.) • Metadata : par ajouter des informations (name, labels, etc.). • spec : pour spécifier les conteneurs à exécuter dans le pod • Voici un exemple d’un Manifest basique (pod-exemple.yaml) : • Le Pod précédent peut être créé par la commande kubectl suivante : apiVersion: v1 kind: Pod metadata: name: nginx spec: containers: - name: nginx image: nginx:1.14.2 ports: - containerPort: 80 kubectl apply -f pod-exemple.yaml
